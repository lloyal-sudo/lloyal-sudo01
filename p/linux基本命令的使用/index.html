Linux基本命令的使用

2026-03-14

一、文件 / 目录操作（最核心）

补充：
查看当前登录用户的用户名

whoami

查看网络配置信息

ip addr
ipconfig

查看进程

ps aux

查找进程中USER为root的进程

ps -u root

输出ps aux的第一列数列

ps aux | awk ‘{print $1}’

通过名字在当前目录下查找新建文件

find . -name “test.txt”

主机ssh虚拟机

ssh 用户名@虚拟机IP

添加用户

sudo useradd 用户名

切换用户

sudo 用户名

在当前用户使用root权限执行命令

sudo 命令

1. 查看当前目录 & 切换目录

核心命令：pwd（显示当前路径）、cd（切换目录），实操：

查看当前工作目录（比如输出 /home/你的用户名）:

pwd

切换到系统根目录:

cd ~
cd

回到上一级目录:

cd ..

切换到具体目录（比如 /usr/local）:

cd /usr/local

2. 查看目录内容

核心命令：ls（列出文件 / 目录），实操任务：

查看当前目录的文件/目录（简略版）:

ls

查看详细信息（权限、大小、修改时间等）:

ls -l

显示隐藏文件（以 . 开头的文件，比如 .bashrc）:

ls -a

组合使用：详细显示所有文件（包括隐藏）:

ls -la

查看指定目录的内容（比如 /var/log）:

ls -l /var/log

3. 创建 / 删除文件 / 目录

核心命令：mkdir（建目录）、touch（建空文件）、rm（删除），实操任务：

在当前目录创建 test 目录:

mkdir test

递归创建多级目录（比如 test/docs/note）:

mkdir -p test/docs/note

在 test 目录下创建空文件 readme.md:

touch test/readme.md

删除空目录 test/docs/note:

rmdir test/docs/note

删除文件 test/readme.md（需确认，加 -f 强制删除）:

rm test/readme.md

强制删除（无提示）:

rm -f test/readme.md

删除非空目录 test（递归删除，谨慎使用！）:

rm -rf test

4. 复制 / 移动文件

核心命令：cp（复制）、mv（移动 / 重命名）实操任务：

先准备测试文件:

touch demo.txt
mkdir temp

复制 demo.txt 到 temp 目录:

cp demo.txt temp/

复制目录（需加 -r 递归）:

mkdir src
touch src/file1.txt
cp -r src/ temp/

重命名文件（把 demo.txt 改成 test.txt）:

mv demo.txt test.txt

移动 test.txt 到 temp 目录:

mv test.txt temp/

移动并改名（把 temp/test.txt 移到当前目录，改名为 new.txt）:

mv temp/test.txt ./new.txt

二、文件内容查看

核心命令：cat、more、less、head、tail，实操任务：

先创建一个有内容的文件:

echo “第一行内容
第二行内容
第三行内容
…
第十行内容” > content.txt

查看文件全部内容:

cat content.txt

查看文件并显示行号:

cat -n content.txt

查看大文件（分页显示，按空格翻页，q 退出）:

more content.txt

更灵活的分页（支持上下翻页）:

less content.txt

查看文件前 5 行（默认前10行）:

head -5 content.txt

查看文件最后 3 行:

tail -3 content.txt

实时监控文件更新（比如日志文件）:

tail -f /var/log/syslog # 按 Ctrl+C 退出

三、权限管理

核心命令：chmod（修改权限），实操任务：

查看文件权限（比如 content.txt）:

ls -l content.txt

#输出示例：-rw-r–r– 1 user user 50 Mar 14 10:00 content.txt

#解读：-rw-r–r– = 所有者可读可写，组用户可读，其他用户可读

给文件所有者添加执行权限:

chmod u+x content.txt

移除其他用户的读权限:

chmod o-r content.txt

用数字权限（最常用：755 目录，644 文件）:

chmod 644 content.txt # 所有者读写，其他只读
chmod 755 temp/ # 所有者读写执行，其他读执行

四、系统信息查看

核心命令：top、df、free、uname，实操任务：

查看系统进程（实时监控，按 q 退出）:

top

查看磁盘空间使用情况（-h 人性化显示）:

df -h

查看内存使用情况（-h 人性化显示）:

free -h

查看系统内核版本:

uname -r

查看系统完整信息:

uname -a

五、文件查找

核心命令：find、grep，实操任务：

在当前目录查找 .txt 后缀的文件:

find . -name “*.txt”

在 /var/log 目录查找修改时间7天内的文件:

find /var/log -mtime -7

在 content.txt 中查找包含「第二行」的行:

grep “第二行” content.txt

忽略大小写查找:

grep -i “第二行” content.txt

显示匹配行的行号：

grep -n “第二行” content.txt

ls -l的命令展示的内容详情

执行ls -l后的输出示例：

-rw-r–r– 1 root root 1234 Mar 14 10:25 demo.txt
drwxr-xr-x 2 user user 4096 Mar 13 18:00 test_dir

以第一行为例，逐列拆解：

-rw-r–r– ：文件类型 + 权限
1 :硬链接数
root :所有者（属主）
root :所属组（属组）
1234 :文件大小
Mar 14 10:25 :最后修改时间
demo.txt :文件名 / 目录名

第一列 10 个字符中的第一个字符：文件类型 - ：普通文件 ，如.txt、.sh、.log 等
d ：目录 ，文件夹
l ：软链接（快捷方式） ，类似 Windows 快捷方式
c ：字符设备文件 ，如键盘、鼠标
b ：块设备文件 ，如硬盘、U 盘

第 2-10 个字符：权限（9 位，分 3 组）
9 位权限分为 3 组（每组 3 位），对应：所有者 (u) → 所属组 (g) → 其他用户 (o)
每组包含：r（读，4）、w（写，2）、x（执行，1）、-（无权限）
示例：rw-r–r– 拆解
所有者（u）：rw- → 可读、可写、不可执行
所属组（g）：r– → 可读、不可写、不可执行
其他用户（o）：r– → 可读、不可写、不可执行

总结：
1.ls -l 输出共 9 列核心信息，核心关注文件类型 + 权限、所有者 / 所属组、文件大小、修改时间。
2.第一列 10 个字符(-rw-r–r–)：第 1 位是文件类型，后 9 位是 3 组权限（所有者 / 所属组 / 其他用户）。

touch命令怎么创造文件管道符的使用

一、touch 命令（创建文件 / 修改时间戳）

touch 最核心的作用是创建空文件，也能修改文件的访问 / 修改时间戳，先看最常用的实操场景：

1. 基础用法：创建空文件

创建单个空文件

touch test.txt

同时创建多个空文件（空格分隔）

touch file1.txt file2.md file3.sh

在指定目录创建文件（目录需先存在）

touch /home/user/docs/note.txt

创建带特殊字符的文件（用引号包裹，不推荐，但需了解）

touch “my file.txt” # 文件名含空格

2.进阶用法：修改文件时间戳（不改变文件内容）

先创建测试文件

touch demo.txt

查看文件的时间戳（ls -l 看修改时间）

ls -l demo.txt

更新文件的「访问时间」和「修改时间」为当前时间

touch demo.txt

只修改「修改时间」（-m）为指定时间（格式：YYYYMMDDhhmm）

touch -m -t 202603141530 demo.txt # 改为2026年3月14日15:30

如果文件不存在，touch 会创建；如果存在，只更新时间戳

touch existing_file.txt # 仅更新时间，不删内容

二、管道符 | 的核心使用

管道符的作用是：把前一个命令的输出，作为后一个命令的输入，是 Linux 中最常用的 “命令拼接” 方式，先看基础逻辑：

命令1 | 命令2 # 命令1的输出 → 命令2的输入

1. 基础实操任务（结合 touch 或常用命令）

筛选 ls 输出，只看 .txt 文件

ls -l | grep “.txt” # ls -l 的输出 → grep 筛选含 .txt 的行

先创建文件，再统计数量

先批量创建5个文件

touch doc{1..5}.txt

用管道统计当前目录下 .txt 文件的数量

ls -l | grep “.txt” | wc -l # wc -l 统计行数（即文件数）

把命令输出写入文件（结合 touch + 管道 + >）

先创建空文件

touch system_info.txt

把系统信息（uname -a）通过管道筛选后写入文件

uname -a | grep “Linux” > system_info.txt # > 是重定向，和管道常配合

查看写入结果

cat system_info.txt

实时监控日志，只看含 “error” 的行

先创建测试日志文件

touch app.log
echo “info: 启动成功” » app.log
echo “error: 连接失败” » app.log

用管道筛选日志中的错误行

cat app.log | grep “error” # 只输出含 error 的行

2. 管道符的核心规则

管道只传递标准输出，不传递错误输出（如果要传错误输出，需用 2>&1，比如 ls /xxx 2>&1 | grep “No such file”）。 管道可以无限拼接：命令1 | 命令2 | 命令3 | …，比如 ls | grep “.txt” | sort | head -2（筛选 txt 文件→排序→取前 2 个）。

3. 常用管道组合示例

组合命令:ps aux ，作用 ：grep "python" ，查找所有 python 进程
组合命令:cat file.txt ，作用 ：wc -l ，统计文件的行数
组合命令:ls -lh ，作用 ：sort -k5 ，按文件大小排序显示（第 5 列是大小）

三、touch + 管道符 综合实战

批量创建10个文件（test1-test10.txt）

touch test{1..10}.txt

筛选含 test 的文件 → 统计数量 → 写入 result.txt

ls -l | grep “test” | wc -l > result.txt

查看结果

cat result.txt # 输出 10

mv如何重命名文件

一、mv 重命名的核心逻辑

mv 原本是 “移动文件 / 目录” 的命令，重命名本质是将文件移动到 “原目录 + 新文件名”，核心语法：

mv 原文件名 新文件名 # 同一目录下 → 重命名
mv 原路径/原文件名 新路径/新文件名 # 跨目录 → 移动+重命名

二、基础实操：单个文件重命名

1. 同一目录下最简单的重命名

先创建测试文件

touch old_name.txt

重命名为 new_name.txt

mv old_name.txt new_name.txt

验证结果（查看文件是否存在）

ls -l new_name.txt # 输出 -rw-r–r– … new_name.txt

2. 带路径的文件重命名

在指定目录创建文件

touch /home/user/docs/old_doc.md

重命名该文件（保持路径不变，只改名字）

mv /home/user/docs/old_doc.md /home/user/docs/new_doc.md

验证

ls -l /home/user/docs/new_doc.md

3. 重命名同时移动目录（移动 + 改名）

先创建测试文件和目标目录

touch file.txt
mkdir new_dir

将 file.txt 移动到 new_dir 并改名为 moved_file.txt

mv file.txt new_dir/moved_file.txt

验证（查看 new_dir 下的文件）

ls -l new_dir/ # 输出 -rw-r–r– … moved_file.txt

三、批量重命名文件

先批量创建测试文件

touch file{1..5}.txt

用 for 循环批量重命名
for i in {1..5}; do
    mv file${i}.txt doc${i}.txt
done

验证结果

ls -l doc*.txt # 输出 doc1.txt doc2.txt … doc5.txt

四、避坑技巧 & 注意事项

1.覆盖警告：如果新文件名已存在，mv 会直接覆盖（无提示），非常危险！

解决方案：加 -i 参数（交互式），覆盖前会提示确认：
创建两个文件：

touch a.txt b.txt

重命名 a.txt 为 b.txt（会提示是否覆盖）

mv -i a.txt b.txt

输出：mv: overwrite ‘b.txt’? （输入 y 覆盖，n 取消）

2.重命名目录：mv 重命名目录和文件用法完全一样：

mkdir old_dir
mv old_dir new_dir # 目录重命名

3.特殊字符处理：如果文件名含空格 / 特殊符号（如 my file.txt），用引号包裹：

touch “my file.txt”
mv “my file.txt” “my new file.txt”

五、实用示例：重命名日志文件（带时间戳）

创建测试日志

touch app.log

获取当前日期（格式：20260314）

DATE=$(date +%Y%m%d)

重命名为 app_20260314.log

mv app.log app_${DATE}.log

验证

ls -l app_${DATE}.log

如何创建软链接并正常使用

一、软链接核心概念

软链接（Symbolic Link）相当于 Windows 的 “快捷方式”：
  它是一个独立文件，指向原文件 / 目录的路径
  删除软链接不影响原文件，删除原文件则软链接失效
  软链接可以跨文件系统，支持指向目录或文件

二、创建软链接：ln -s 命令

ln -s 原文件/目录的路径 软链接的路径

给文件创建软链接

先创建测试文件（原文件）

touch /home/user/docs/note.txt
echo “测试内容” > /home/user/docs/note.txt

创建软链接（把 note.txt 的快捷方式放到用户主目录）

ln -s /home/user/docs/note.txt /home/user/note_link.txt

验证软链接是否创建成功（ls -l 看标识）

ls -l /home/user/note_link.txt

输出示例（l 表示软链接，-> 指向原文件）：
lrwxrwxrwx 1 user user 24 Mar 14 16:00 note_link.txt -> /home/user/docs/note.txt

给目录创建软链接（更常用）

先创建测试目录（原目录）

mkdir -p /home/user/data/logs

创建目录的软链接（把 logs 目录的快捷方式放到 /tmp 下）

ln -s /home/user/data/logs /tmp/logs_link

验证

ls -l /tmp/ | grep logs_link

输出示例：
lrwxrwxrwx 1 user user 20 Mar 14 16:05 logs_link -> /home/user/data/logs/

三、正常使用软链接（读写 / 进入 / 操作）

软链接的使用和原文件 / 目录完全一致，无需特殊操作：

1. 使用文件软链接

读取软链接内容（等同于读原文件）

cat /home/user/note_link.txt # 输出：测试内容

修改软链接内容（实际修改的是原文件）

echo “新增内容” » /home/user/note_link.txt
cat /home/user/docs/note.txt # 能看到新增内容：测试内容\n新增内容

2. 使用目录软链接

进入软链接目录（等同于进入原目录）

cd /tmp/logs_link

在软链接目录创建文件（实际创建在原目录）

touch /tmp/logs_link/app.log
ls /home/user/data/logs/ # 能看到 app.log

查看目录软链接的内容

ls -l /tmp/logs_link/

四、关键注意事项（避坑！）

1. 路径问题（最容易出错）

推荐用绝对路径：创建软链接时，原文件 / 目录和软链接都用绝对路径（如 /home/user/xxx），避免软链接移动后失效。
若用相对路径：仅当软链接和原文件的相对位置不变时才有效，否则会报错 No such file or directory。

2. 原文件 / 目录被删除 / 移动后的处理

假设删除原文件

rm /home/user/docs/note.txt

此时访问软链接会报错

cat /home/user/note_link.txt
输出：cat: /home/user/note_link.txt: No such file or directory

解决：恢复原文件到原路径，或重新创建软链接指向新路径

ln -sf /新路径/note.txt /home/user/note_link.txt # -f 强制覆盖旧软链接

3. 权限问题

软链接的权限始终是 lrwxrwxrwx（所有用户读写执行），但实际访问权限由原文件 / 目录决定。
例：原文件是 rw-r–r–，即使软链接权限是 rwxrwxrwx，其他用户也只能读，不能写。

五、删除软链接（避免误删原文件）

1. 删除文件软链接（直接 rm 软链接名，不加 /）

正确删除（不加 /，否则会删原文件！）

rm /home/user/note_link.txt

错误示范（加了 /，会尝试删除原文件，幸运的是会提示）

rm /home/user/note_link.txt/
输出：rm: cannot remove ‘/home/user/note_link.txt/’: Not a directory

2. 删除目录软链接（同样不加 /）

正确删除

rm /tmp/logs_link

或用 rm -rf（但仍不加 /！）

rm -rf /tmp/logs_link
⚠️ 关键：删除目录软链接时，绝对不要在软链接名后加 /，否则会递归删除原目录的内容！

六、常见问题排查

问题 1：软链接显示红色 / 闪烁，提示失效
原因：原文件 / 目录被删除、移动，或软链接的路径写错。
解决：检查原文件路径是否存在，重新创建软链接。
问题 2：cd 进软链接目录后，pwd 显示软链接路径而非原路径
原因：默认行为，可加 -P 参数显示原路径：

cd -P /tmp/logs_link
pwd # 输出原目录路径：/home/user/data/logs

如何复制文件到指定位置

一、cp 命令核心语法

复制文件的基础格式：

cp [ 可选参数 ] 源文件路径 目标位置路径

核心逻辑：
源文件路径：要复制的文件（绝对 / 相对路径都可）
目标位置路径：可以是目录（文件会复制到该目录下，文件名不变），也可以是新文件名（复制并重命名）

二、基础实操：复制文件到指定位置

场景 1：复制文件到指定目录（文件名不变）

先创建测试文件和目标目录

touch /home/user/test.txt # 源文件
mkdir -p /home/user/backup # 目标目录

复制 test.txt 到 backup 目录（最基础用法）

cp /home/user/test.txt /home/user/backup/

验证是否复制成功

ls -l /home/user/backup/test.txt
#输出：-rw-r–r– 1 user user 0 Mar 14 17:00 /home/user/backup/test.txt

场景 2：复制文件到指定目录并改名

复制 test.txt 到 backup 目录，并重命名为 test_bak.txt

cp /home/user/test.txt /home/user/backup/test_bak.txt

验证

ls -l /home/user/backup/test_bak.txt

场景 3：复制多个文件到指定目录

创建多个测试文件

touch file1.txt file2.md file3.sh

一次性复制这3个文件到 backup 目录

cp file1.txt file2.md file3.sh /home/user/backup/

验证

ls -l /home/user/backup/ # 能看到3个文件

场景 4：用相对路径复制（更灵活）

假设当前目录是 /home/user
源文件相对路径：./test.txt（当前目录下的 test.txt）
目标目录相对路径：./backup/（当前目录下的 backup 目录）

cp ./test.txt ./backup/

三、进阶用法：处理覆盖 / 保留属性

1. 安全复制（覆盖前提示）

默认 cp 会直接覆盖目标文件（无提示），加 -i 参数（interactive）可避免误覆盖：

先在 backup 目录创建同名文件

touch /home/user/backup/test.txt

复制时提示是否覆盖

cp -i /home/user/test.txt /home/user/backup/
输出：cp: overwrite ‘/home/user/backup/test.txt’? （输入 y 覆盖，n 取消）

2. 强制复制（无提示覆盖）

加 -f 参数（force），跳过提示直接覆盖（适合脚本批量操作）：

cp -f /home/user/test.txt /home/user/backup/

3. 复制时保留文件属性（权限、时间戳等）

加 -p 参数（preserve），适合备份场景，保留源文件的权限、所有者、修改时间等：

先给源文件加执行权限

chmod +x /home/user/test.txt

带属性复制

cp -p /home/user/test.txt /home/user/backup/

验证：目标文件和源文件权限一致

ls -l /home/user/test.txt ls -l /home/user/backup/test.txt

四、扩展：复制目录（递归复制）

如果需要复制整个目录（含子文件 / 子目录），加 -r 参数（recursive）：

创建测试目录和子文件

mkdir -p /home/user/docs/note
touch /home/user/docs/note/readme.md

递归复制 docs 目录到 backup 目录

cp -r /home/user/docs /home/user/backup/

验证：子文件也被复制

ls -l /home/user/backup/docs/note/readme.md

五、避坑技巧

1.目标路径结尾加 / 更安全：复制到目录时，目标路径最后加 /，明确表示 “复制到该目录下”，避免误改名：

正确（复制到 backup 目录下）

cp test.txt /home/user/backup/

错误（如果 backup 不是目录，会把 test.txt 重命名为 backup）

cp test.txt /home/user/backup

2.处理空格 / 特殊字符：文件名含空格时，用引号包裹路径：

touch “my file.txt”
cp “my file.txt” /home/user/backup/

3.跨用户复制：如果目标目录属于其他用户，加 sudo 提升权限：

sudo cp test.txt /root/backup/

六、实用示例：日常备份脚本

脚本功能：每天备份 /home/user/logs 下的 .log 文件到 /tmp/backup_logs!/bin/bash
创建目标目录

mkdir -p /tmp/backup_logs 复制所有 .log 文件，保留属性，覆盖前提示 cp -ip /home/user/logs/.log /tmp/backup_logs/* 验证 echo “备份完成，文件列表：”
ls -l /tmp/backup_logs/

绝对路径和相对路径的辨识

绝对路径是 “全局地址”，相对路径是 “局部地址”

一、实操案例（对比理解）

先搭建测试目录结构，方便演示：
创建目录结构：/home/user → docs → note → test.txt

mkdir -p /home/user/docs/note
touch /home/user/docs/note/test.txt

场景 1：当前目录是 /home/user（根出发的第 3 层）

先切换到当前目录

cd /home/user

用绝对路径访问 test.txt（无论在哪，绝对路径都能找到）

cat /home/user/docs/note/test.txt

用相对路径访问 test.txt（从当前目录出发）

cat docs/note/test.txt # 相对路径：当前目录 → docs → note → test.txt

场景 2：当前目录是 /home/user/docs/note（目标文件同目录）

切换目录

cd /home/user/docs/note

绝对路径（依然不变）

cat /home/user/docs/note/test.txt

相对路径（最简，直接写文件名）

cat test.txt

场景 3：当前目录是 /home/user/docs（目标文件在子目录）

切换目录

cd /home/user/docs

绝对路径（不变）

cat /home/user/docs/note/test.txt

相对路径（当前目录 → note → test.txt）

cat note/test.txt

反向相对路径（比如从 note 回到 docs）

*cd note # 进入 note 目录
*cd .. # .. 表示「上一级目录」，回到 docs
*cd ~ # 快速回到用户主目录（绝对路径的快捷方式）
*cd . # . 表示「当前目录」，无实际作用，仅作标识

二、常用相对路径符号（必记）

符号：. ，含义：当前目录 ，示例：cp ./test.txt /tmp（复制当前目录的 test.txt）
符号：.. ，含义：上一级目录 ，示例：cd ..（回到上一级）
符号：~ ，含义：用户主目录（快捷） ，示例：cd ~/docs（等价于 /home/user/docs）
符号：- ，含义：上一次所在目录 ，示例：cd -（切换回上一次的目录）

三、实战场景

1. 用绝对路径的场景（推荐优先用）

编写脚本：脚本运行时当前目录不确定，绝对路径能保证文件找到（比如备份脚本 cp /home/user/logs/app.log /tmp/backup/）。
创建软链接：之前讲过，软链接用绝对路径不会失效（ln -s /home/user/docs/test.txt /tmp/test_link.txt）。
系统配置：配置文件中指定路径（比如 Nginx 配置 root /var/www/html;）。

2. 用相对路径的场景

日常操作：当前目录层级深，简短路径更方便（比如 cd ../note 比 cd /home/user/docs/note 快）。
批量操作：同目录下处理多个文件（比如 cp *.txt ../backup/）。

cat：一次性看完（小文件）

cat /var/log/dmesg

用常用的二进制文件目录给文件赋权

给文件赋权：chmod 命令（必考必会）

Linux 文件有 3 种权限：
r 读 → 数字 4
w 写 → 数字 2
x 执行 → 数字 1

分为三类人：
u 文件所有者（自己）
g 同组用户
o 其他人

最简单：数字赋权（最常用）

chmod 数字权限 文件名

最常用的 3 个权限：
644：只读文件（文本、配置）
自己：读写 (6)，别人：只读 (4)

chmod 644 test.txt

755：可执行文件（脚本、二进制、命令）
自己：读写执行 (7)，别人：读执行 (5)

chmod 755 my.sh

777：谁都能改（不推荐，仅测试用）

chmod 777 test.txt

运行可执行文件和递归删除

一、运行可执行文件（3 种常用方式）

可执行文件包括：二进制文件、Shell 脚本、Python 脚本等（前提是用 chmod 加了执行权限，比如 chmod 755 文件名）。

1. 最简单：相对路径运行（最常用）

./文件名

./ 表示「当前目录」，必须加！否则系统会去 /bin 等系统目录找，找不到就报错。
实操示例：
先创建一个可执行脚本（加执行权限）

echo “echo ‘我是可执行文件’” > test.sh
chmod 755 test.sh

运行（当前目录下）

./test.sh # 输出：我是可执行文件

2. 绝对路径运行（精准，适合脚本 / 配置）

/完整路径/文件名

示例：
假设 test.sh 在 /home/user 目录下

/home/user/test.sh

3. 直接运行（仅系统目录下的文件）

如果可执行文件在 /bin、/usr/bin 等系统默认目录里，直接敲文件名即可（不用加路径）：

ls # 等价于 /bin/ls
cp # 等价于 /bin/cp

二、递归删除（删除目录 + 所有子文件 / 子目录）

核心命令：rm -rf（必须谨慎用！删了很难恢复）
-r（recursive）：递归删除（目录 + 子内容）
-f（force）：强制删除，无提示（避免删目录时弹一堆确认）

1. 递归删除目录（最常用）

rm -rf 目录名

实操示例：
先创建带内容的测试目录

mkdir -p test_dir/sub_dir
touch test_dir/file1.txt test_dir/sub_dir/file2.txt

2. 递归删除整个 test_dir（包括子目录和文件）

rm -rf test_dir

3. 验证：目录已消失

ls -l | grep test_dir # 无输出，说明删干净了

2. 递归删除指定类型文件（比如所有 .log 文件）

rm -rf 目录名/.log* 示例：

使用find、grep查找flag在哪些文件中

一、最常用：全局查找含 flag 的文件

1. 基础命令（直接用）

从根目录 / 开始，查找所有文件中包含 flag 的内容（显示文件名+匹配行）

find / -type f -exec grep -l “flag” {} ;

2. 实操示例（更实用）

场景 1：只在 /home/user 目录找，排除系统目录（更快）

find /home/user -type f -exec grep -l “flag” {} ;

场景 2：显示文件名 + 具体匹配行（更直观）
把 -l 换成 -n（显示行号），能看到 flag 出现在文件的第几行：

find /home/user -type f -exec grep -n “flag” {} ;

输出示例：

/home/user/test.txt:5:this is the flag: abc123
/home/user/note.md:10:flag=def456

场景 3：忽略大小写查找（比如 Flag/FLAG 也能搜到）

find /home/user -type f -exec grep -il “flag” {} ;

-i：忽略大小写。

二、进阶：缩小查找范围（更快）

如果知道文件类型（比如只找 .txt/.sh 文件），可以限定后缀，避免搜无关文件：
只找 /home/user 下的 .txt 文件，搜包含 flag 的

find /home/user -type f -name “.txt” -exec grep -l “flag” {} ;*

-name “*.txt”：只匹配后缀为 .txt 的文件。

三、简化写法（管道符组合）

也可以用 find 输出文件列表，再通过管道传给 grep，效果一样：

*find /home/user -type f | xargs grep -l “flag”

xargs：把 find 找到的文件列表传给 grep（替代 -exec，更简洁）。

四、避坑技巧

权限不足报错：搜系统目录（如 /）时会报 Permission denied，可以加 2>/dev/null 屏蔽错误：

find / -type f -exec grep -l “flag” {} ; 2>/dev/null

只搜最近修改的文件：比如找 7 天内修改的文件，加 -mtime -7：

find /home/user -type f -mtime -7 -exec grep -l “flag” {} ;

避免搜大文件 / 二进制文件：可以限定文件大小（比如小于 100M），加 -size -100M：

find /home/user -type f -size -100M -exec grep -l “flag” {} ;
