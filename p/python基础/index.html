一、初识Python

1. 什么是 Python？

  • 一种解释型、面向对象、高级编程语言，语法简洁，接近自然语言，适合新手入门。
  • 应用场景：数据分析、人工智能、Web 开发、自动化脚本、爬虫等。
  • 核心优势：上手快、第三方库丰富（无需重复造轮子）、跨平台（Windows/Mac/Linux 都能用）。

二、变量

变量是 Python 编程的基础核心，相当于“存储数据的容器”，掌握变量的定义、使用和特性，是入门 Python 的关键第一步。以下整理从基础概念到进阶用法的完整知识点，适配新手理解节奏。

变量的核心概念

1. 什么是变量？

  • 通俗理解：变量就像一个贴了标签的盒子，盒子里可以放数字、文字、布尔值等数据，标签就是变量名，通过标签能快速找到盒子里的内容。
  • 专业定义：程序运行过程中可以改变的量，用于存储和表示数据，Python 中变量无需提前声明类型（动态类型语言）。
  • 核心特点：
    • 变量的值可以随时修改；
    • 一个变量只能存储一个值（新值会覆盖旧值）。

示例：

1
2
3
4

# 给变量赋值，相当于往“盒子”里放数据
score = 90  # 变量score存储整数90
score = 95  # 新值95覆盖旧值90，现在score的值是95
print(score)  # 输出：95

2. 变量的赋值语法

Python 中用 = 赋值（不是“等于”，是“赋值”），语法格式：
变量名 = 数据

  • 等号左边：变量名（自定义的标签）；
  • 等号右边：要存储的数据（可以是直接值、表达式、其他变量）。

b = a + 5       # 表达式赋值：变量b存储a+5的结果15
c = b           # 变量赋值：变量c存储b的值15
print(a, b, c)  # 输出：10 15 15

变量的命名规则（必须遵守）

Python 对变量名有严格规范，违反会直接报错，核心规则如下：

1. 合法规则（必须满足）

  1. 由字母（大小写）、数字、下划线组成；
  2. 不能以数字开头；
  3. 不能使用 Python 的关键字（内置的保留字，有特殊用途）；
  4. 区分大小写（name 和 Name 是两个不同变量）。

2. 关键字列表（新手只需眼熟，避免使用）

  2. 多单词命名：
    • 下划线法（Python 官方推荐）：student_name、user_age；

  1. 除数为0：10 / 0 会报错 ZeroDivisionError，需避免；
  2. 类型不匹配运算：10 + "5" 报错 TypeError，需先转换类型（10 + int("5")）；
  3. 混淆 = 和 ==：if age = 18: 报错（赋值不能用于条件），应写 if age == 18:；
  4. 逻辑运算符滥用：判断区间 10 < score < 20 等价于 score >10 and score <20，新手直接写前者（错误）更简洁。

总结

  1. Python 运算符核心分4类：算术运算符（数值计算）、赋值运算符（变量赋值）、比较运算符（条件判断）、逻辑运算符（组合条件）；
  2. 比较/逻辑运算符的结果都是布尔值（True/False），是 if/while 的核心；
  3. 不确定运算符优先级时，用 () 括号明确执行顺序，提升代码可读性。

if 语句是 Python 中实现条件判断的核心语法，也是编程中“逻辑分支”的基础——让程序根据不同条件执行不同的代码块。以下从基础概念到进阶用法，系统化整理新手必掌握的知识点。

if 语句的核心概念

1. 什么是 if 语句？

  • 通俗理解：程序的“选择题”——满足条件就执行A操作，不满足就执行B操作（或不执行）。

五、循环语句

循环语句是 Python 实现重复执行一段代码的核心语法，能解决“需要多次做相同/相似操作”的问题（比如计算1-100的和、遍历列表所有元素）。以下从基础概念到进阶用法，系统化整理新手必掌握的知识点。

循环语句的核心概念

1. 什么是循环？

  • 通俗理解：程序的“重复操作”——比如每天重复吃饭、睡觉，对应代码中重复执行某段逻辑。
  • 核心分类：Python 中循环主要分两类：
    • for 循环：遍历序列（字符串、列表、数字范围等），执行次数由序列长度决定；
    • while 循环：条件满足就执行，执行次数由条件是否成立决定（可能无限循环）。
  • 核心特点：
    • 循环体（重复执行的代码）需用缩进（4个空格/1个Tab）划分；
    • 可通过 break/continue 控制循环流程。

七、列表&元组

列表（list）—— 可变的有序序列

1. 核心概念

  • 通俗理解：像“可修改的购物清单”，可以添加、删除、修改里面的物品，元素按顺序排列。
  • 定义标识：用 [] 包裹，元素之间用 , 分隔，支持存储不同类型的数据（数字、字符串、列表等）。
  • 核心特点：可变（可修改元素、增删元素）、有序（支持索引、切片）、可重复（允许元素重复）。

八、字典&集合

字典（dict）和集合（set）是 Python 中处理非序列型数据的核心容器类型——字典主打“键值对”映射，集合主打“无序唯一”，二者都是新手入门后突破“有序序列思维”的关键知识点。以下从基础概念、核心操作、异同对比到实战场景，系统化整理新手必掌握的内容。

九、类型转换 & 深浅拷贝

类型转换是 Python 处理不同数据类型交互的基础，深浅拷贝则是处理“可变对象赋值”的关键（避免新手踩“修改副本影响原对象”的坑）。以下从基础概念、核心用法、常见场景到避坑指南，系统化整理新手必掌握的知识点。

类型转换（数据类型的“变身术”）

1. 核心概念

  • 通俗理解：把一种数据类型转换成另一种（比如把用户输入的字符串“18”变成数字18，才能做加法运算）。
  • 核心分类：
    • 显式转换（手动转换）：用 int()/float()/str() 等函数主动转换，新手必掌握；
    • 隐式转换（自动转换）：Python 底层自动完成（如 1 + 2.5 自动转成浮点数运算），无需手动干预。
  • 核心原则：只有“兼容”的类型才能转换（比如纯数字字符串可转数字，含字母的字符串不行）。

1. 前置概念：Python 的“赋值”≠“复制”

新手常踩坑：直接用 = 赋值可变对象（列表/字典），修改“副本”会影响原对象，因为 = 只是“引用指向”，而非真正复制。

  1. 仅需使用数据，无需修改 → 直接赋值（=）即可，节省内存；
  2. 需修改副本，但对象无嵌套（如单层列表/字典）→ 浅拷贝；
  3. 需修改副本，且对象含嵌套（如列表里套列表、字典里套列表）→ 深拷贝。

十、函数基础

函数是 Python 中封装可复用代码的核心机制，也是结构化编程的基础——把重复执行的逻辑封装成函数，需要时直接调用，无需重复写代码。以下从基础概念、定义/调用、参数、返回值到实战场景，系统化整理新手必掌握的函数知识点。

函数的核心概念

1. 什么是函数？

  • 通俗理解：像“预制菜的配方”——提前定义好步骤（代码逻辑），需要时按配方（调用函数）就能做出菜（执行逻辑），可反复使用。
  • 专业定义：组织好的、可重复使用的、实现特定功能的代码块。
  • 核心价值：
    • 代码复用：避免重复写相同逻辑；
    • 简化代码：将复杂逻辑拆分成多个函数，结构更清晰；
    • 易维护：修改函数即可同步更新所有调用处的逻辑。

异常处理（Error Handling）

1. 核心概念

  • 通俗理解：程序运行时出现的“错误”（比如除数为0、打开不存在的文件、索引越界），若不处理会导致程序崩溃。
  • 专业定义：Python中程序执行过程中发生的不正常情况（偏离预期的事件），分为两类：

    • 运行时异常：语法正确但运行时出错（如1/0），需用异常处理机制捕获。
  • 核心价值：捕获异常并给出友好处理（而非直接崩溃），让程序“容错”。

常见错误与避坑指南

1. 异常处理

  • 滥用except Exception：精准捕获具体异常（如ZeroDivisionError），而非兜底捕获所有异常，否则会隐藏真正的问题；
  • 捕获异常后无处理：至少打印异常信息，避免“吞掉异常”导致排查困难。

2. 模块/包

  • 模块命名冲突：自定义模块名不要和Python内置模块重名（如不要命名为math.py），否则会覆盖内置模块；
  • 包导入路径错误：确保包的目录在Python的搜索路径中（新手建议将包放在当前项目根目录）；
  • 第三方包安装失败：检查pip版本、网络，或切换镜像源（如pip install -i https://pypi.tuna.tsinghua.edu.cn/simple requests）。

总结

  1. 异常处理：核心用try-except捕获运行时错误，精准捕获具体异常，finally用于释放资源，raise主动抛异常；
  2. 模块：一个.py文件就是模块，用import导入，__name__ == "__main__"控制模块执行逻辑；
  3. 包：含__init__.py的文件夹，用于组织多个模块，第三方包用pip安装后导入使用；
  4. 核心原则：异常处理让程序更健壮，模块/包让代码更易复用和维护。

十二、面向对象基础

面向对象（OOP）是 Python 编程的核心范式之一，相比“面向过程”（按步骤写代码），面向对象更贴近现实世界的思维方式——将数据和操作数据的方法封装成“对象”，通过类的抽象和复用提升代码的可维护性。以下从核心概念、类与对象、三大特性到实战案例，系统化整理新手必掌握的面向对象基础知识点。

面向对象的核心概念

1. 核心术语（类比现实世界）

术语 通俗理解 现实例子 Python 对应
类（Class） 事物的“模板/蓝图”，定义属性和行为 汽车设计图纸（规定汽车有颜色、能行驶） class Car: 定义的类
对象（Object） 类的“实例”，具体的事物 张三的黑色特斯拉（符合图纸的具体汽车） my_car = Car("黑色") 创建的实例
属性（Attribute） 事物的“特征/数据” 汽车的颜色、品牌、速度 类中的变量（如 self.color）
方法（Method） 事物的“行为/操作” 汽车的行驶、刹车、鸣笛 类中的函数（如 def run(self):）

2. 面向对象的三大核心特性（重点）

  • 封装：将数据（属性）和操作数据的方法打包到类中，隐藏内部细节，仅暴露必要接口；
  • 继承：子类复用父类的属性和方法，同时可扩展自己的功能；
  • 多态：不同子类对同一方法有不同实现，调用时自动适配（新手入门先理解概念，后续深入）。

        self.price = price

class ShoppingCart:
    """购物车类：封装购物车行为"""
    def __init__(self):
        self.goods_list = []  # 购物车中的商品列表
    
    # 添加商品到购物车
    def add_goods(self, goods, buy_count):
        if buy_count > goods.count:
            print(f"库存不足！{goods.name}仅剩{goods.count}件")
            return
        # 检查购物车中是否已有该商品
        for item in self.goods_list:
            if item["goods"].name == goods.name:
                item["count"] += buy_count
                goods.count -= buy_count
                print(f"购物车中{goods.name}数量更新为{item['count']}件")
                return
        # 新增商品到购物车
        self.goods_list.append({"goods": goods, "count": buy_count})
        goods.count -= buy_count
        print(f"{goods.name}×{buy_count}已加入购物车")
    
    # 计算购物车总价
    def calc_total(self):
        total = 0
        for item in self.goods_list:
            total += item["goods"].price * item["count"]
        return total
    
    # 展示购物车
    def show_cart(self):
        print("===== 购物车 =====")
        for item in self.goods_list:
            print(f"{item['goods'].name} × {item['count']} 件")
        print(f"总价：{self.calc_total()}元")

十三、封装&单继承

封装和单继承是 Python 面向对象（OOP）中最基础且高频使用的两大特性——封装聚焦“数据安全与接口简化”，单继承聚焦“代码复用与功能扩展”。以下从核心概念、语法规则、实战示例到避坑指南，系统化整理新手必掌握的知识点。

封装（Encapsulation）—— 数据的“保护罩”

1. 核心概念

  • 通俗理解：把事物的属性（数据） 和方法（操作数据的逻辑） 打包到类中，隐藏内部实现细节，仅暴露安全的访问接口（类似手机：用户只需按按钮操作，无需知道内部电路原理）。
  • 核心目的：
    • 数据安全：避免外部直接修改属性导致数据异常（如成绩不能为负数、年龄不能为字符串）；
    • 代码解耦：内部实现修改时，只要接口不变，外部调用无需调整；
    • 简化使用：用户只需关注“怎么用”，无需关注“怎么实现”。

2. 封装的核心实现：私有属性/方法 + 公开接口

Python 中通过“命名规则”实现封装（无严格的访问权限关键字，靠约定）：

命名方式 访问范围 用途
普通名（如 name） 类内/类外均可访问 无需隐藏的通用属性/方法
单下划线（如 _age） 类内可访问，类外不建议访问 弱私有（仅提醒开发者，仍可访问）
双下划线（如 __score） 仅类内可访问，类外无法直接访问 强私有（真正的封装，核心数据）

class Student:
    """学生类：封装核心数据和操作"""
    def __init__(self, name, score):
        self.name = name          # 普通属性（公开）
        self.__score = score      # 双下划线：私有属性（仅类内可访问）
    
    # 私有方法：仅类内调用，处理核心逻辑
    def __check_score(self, new_score):
        """校验成绩是否合法（0-100）"""
        return 0 <= new_score <= 100
    
    # 公开接口：获取私有属性（读）
    def get_score(self):
        return self.__score
    
    # 公开接口：修改私有属性（写，带校验）
    def set_score(self, new_score):
        if self.__check_score(new_score):
            self.__score = new_score
            print(f"{self.name}的成绩更新为：{self.__score}")
        else:
            print(f"错误：成绩{new_score}不合法（需0-100）")
    
    # 公开接口：封装业务逻辑
    def show_info(self):
        print(f"姓名：{self.name}，成绩：{self.__score}")

3. 封装的最佳实践

  1. 核心数据私有化：将需要校验、易出错的属性设为双下划线私有（如成绩、年龄、金额）；
  2. 提供统一接口：为私有属性提供 get_xxx()（读）和 set_xxx()（写）方法，写方法中必须加数据校验；
  3. 弱私有仅作提醒：单下划线（如 _age）仅用于提醒开发者“该属性不建议外部修改”，无需严格校验；
  4. 业务逻辑封装：将复杂的操作逻辑封装到类的方法中，外部只需调用方法，无需关注内部步骤。

单继承（Single Inheritance）—— 代码的“复用器”

1. 核心概念

  • 通俗理解：子类（派生类）继承父类（基类/超类）的所有非私有属性和方法，同时可扩展自己的独有功能（类似儿子继承父亲的财产，同时拥有自己的新资产）。
  • 单继承：一个子类仅继承一个父类（Python 支持多继承，但新手优先掌握单继承，避免复杂度）；
  • 核心目的：
    • 代码复用：父类的通用功能无需在子类中重复编写；
    • 功能扩展：子类在复用父类功能的基础上，新增独有功能或修改原有功能。

常见错误与避坑指南

1. 封装相关

  • 直接修改私有属性：试图通过 obj.__attr 访问私有属性，导致 AttributeError → 解决：仅通过 get/set 接口操作；
  • 接口无数据校验：set_xxx() 方法未加校验，导致非法数据进入 → 解决：所有写接口必须加数据合法性校验；
  • 过度封装：将无需隐藏的属性（如姓名）设为私有，增加代码复杂度 → 解决：仅封装核心敏感数据。

2. 单继承相关

  • 子类重写__init__时未调用super()：导致父类属性未初始化，调用父类方法时报错 → 解决：子类__init__第一行必须调用 super().__init__()；
  • 混淆super()的用法：在非重写方法中滥用super() → 解决：仅在子类需要复用父类逻辑时使用；
  • 子类试图访问父类私有方法/属性：导致报错 → 解决：父类若想让子类复用，应将属性/方法设为单下划线（弱私有），而非双下划线。

总结

核心要点

  1. 封装：

    • 核心是“私有属性+公开接口”，双下划线实现强私有，单下划线仅作提醒；
    • 写接口（set_xxx）必须加数据校验，读接口（get_xxx）按需提供；
    • 目的是保护数据安全、简化外部使用。

  2. 单继承：

    • 核心语法：class 子类(父类):，通过 super() 复用父类逻辑；
    • 子类可重写父类方法实现扩展，可新增独有属性/方法；
    • 父类私有属性/方法子类无法继承，需通过父类公开接口访问。

  3. 组合使用：

    • 父类封装通用数据和逻辑，子类继承后扩展独有功能；
    • 子类的核心数据仍需封装，保证整个类体系的数据安全。

十四、多继承&多态

多继承和多态是 Python 面向对象（OOP）中进阶且易踩坑的两大特性——多继承解决“一个类需要复用多个父类功能”的场景，多态实现“同一接口适配不同对象”的灵活调用。以下从核心概念、语法规则、实战示例到避坑指南，系统化整理新手必掌握的知识点。

多继承（Multiple Inheritance）—— 复用多个父类的能力

1. 核心概念

  • 通俗理解：一个子类同时继承多个父类的非私有属性和方法（类似孩子同时继承父亲的身高、母亲的肤色）；
  • 单继承 vs 多继承：
    • 单继承：class 子类(父类1):（简单、无冲突，新手优先）；

多态（Polymorphism）—— 同一接口，不同实现

1. 核心概念

  • 通俗理解：不同子类对同一父类方法有不同实现，调用时无需关注子类具体类型，只需调用统一接口（类似“按下播放键”：手机、电脑、电视的播放逻辑不同，但操作接口一致）；
  • 核心前提：
    1. 子类继承父类；
    2. 子类重写父类的同名方法；
  • 核心价值：
    • 降低代码耦合度：调用方无需修改逻辑，新增子类即可适配；
    • 提高代码扩展性：新增功能只需新增子类，无需修改原有调用逻辑。

常见错误与避坑指南

1. 多继承相关

  • 盲目使用多继承：导致代码逻辑混乱、难以维护 → 解决：优先单继承 + 组合（如 self.car = Car()），仅在确有必要时使用多继承；
  • 忽略MRO顺序：调用同名方法时预期结果与实际不符 → 解决：通过 类名.mro() 查看方法查找顺序，必要时重写方法；
  • 菱形继承重复初始化：直接调用 父类.__init__(self) 导致顶层父类多次初始化 → 解决：统一使用 super().__init__()。

2. 多态相关

  • 子类未重写父类方法：调用统一接口时执行父类逻辑，不符合预期 → 解决：使用抽象类（abc模块）强制子类重写；
  • 调用方硬编码子类类型：如 if type(animal) == Dog，失去多态优势 → 解决：调用方仅依赖父类接口，不关注子类具体类型；
  • 父类未定义统一接口：子类方法名不一致（如 speak()/bark()），无法统一调用 → 解决：父类定义规范的统一接口名称。

总结

核心要点

  1. 多继承：

    • 语法：class 子类(父类1, 父类2):，方法查找遵循 MRO 顺序（类名.mro() 查看）；
    • 核心问题：方法冲突、菱形继承，优先通过“重写方法+手动调用指定父类”解决；
    • 使用原则：能不用则不用，优先单继承 + 组合，仅在子类需同时复用多个父类核心能力时使用。

  2. 多态：

    • 核心：子类重写父类统一接口，调用方通过父类接口调用，自动适配不同子类实现；
    • 强化：使用 abc 模块定义抽象类，强制子类实现接口，避免漏写；
    • 价值：降低耦合、提高扩展，新增功能只需新增子类，无需修改原有调用逻辑。

  3. 组合使用：

    • 多继承解决“复用多个父类功能”，多态解决“统一接口调用不同实现”；
    • 实际开发中，多态使用频率远高于多继承，是面向对象灵活性的核心体现。

十六、正则表达式

正则表达式（Regex/RE）是处理字符串的“瑞士军刀”——通过一套预定义的语法规则，快速实现字符串的匹配、查找、替换、分割等复杂操作。Python 中通过 re 模块提供正则表达式支持，掌握它能大幅提升字符串处理效率。以下从核心概念、语法规则、常用方法到实战场景，系统化整理新手必掌握的知识点。

正则表达式核心概念

基本定义

  • 通俗理解：一套描述字符串模式的“规则表达式”，用于匹配符合特定格式的字符串（如验证手机号、提取邮箱、过滤敏感词）。
  • 核心价值：
    • 简化复杂字符串操作（如“验证手机号是否合法”，用正则仅需一行规则，无需多层 if-else）；
    • 跨语言通用（正则语法在 Python/Java/JavaScript 等语言中基本一致）。

正则表达式核心语法（新手必背）

正则语法是一套“匹配规则”，分为原子（基础匹配单元）、元字符（特殊功能符号）、限定符（数量控制）三大类。

1. 原子（基础匹配单元）

匹配单个字符的最基本单位，是正则的“最小构成”。

原子类型 语法 说明 示例
普通字符 a/1/中 匹配自身（字母、数字、中文等） abc 匹配 “abc”
转义字符 \d/\w/\s 预定义原子（匹配一类字符） \d 匹配任意数字
自定义原子集 [abc]/[0-9] 匹配集合内任意一个字符 [0-9] 等价于 \d
原子集取反 [^abc] 匹配不在集合内的任意一个字符 [^0-9] 匹配非数字
任意字符 . 匹配除换行符 \n 外的任意一个字符 a.c 匹配 “abc”/“a1c”

语法 说明 等价原子集
\d 匹配任意数字（0-9） [0-9]

\s 匹配任意空白字符（空格/制表符/换行等） [ \t\n\r\f\v]
\S 匹配任意非空白字符 [^ \t\n\r\f\v]

2. 限定符（数量控制）

控制“原子”或“分组”的匹配次数，是实现“多字符匹配”的核心。

限定符 说明 示例
* 匹配前面的原子 0 次或多次 a* 匹配 “”/“a”/“aa”
+ 匹配前面的原子 1 次或多次 a+ 匹配 “a”/“aa”（不匹配 “"）
? 匹配前面的原子 0 次或 1 次 a? 匹配 “”/“a”
{n} 匹配前面的原子恰好 n 次 a{3} 匹配 “aaa”
{n,} 匹配前面的原子至少 n 次 a{2,} 匹配 “aa”/“aaa”
{n,m} 匹配前面的原子 n 到 m 次 a{2,4} 匹配 “aa”/“aaa”/“aaaa”

3. 边界匹配（位置控制）

限定匹配的位置（开头/结尾），避免“部分匹配”问题。

语法 说明 示例
^ 匹配字符串的开头 ^abc 匹配 “abc123”（不匹配 “123abc”）
$ 匹配字符串的结尾 abc$ 匹配 “123abc”（不匹配 “abc123”）
\b 匹配单词边界（非 \w 的位置） \bis\b 匹配 “is”（不匹配 “this” 中的 “is”）
\B 匹配非单词边界 \Bis\B 匹配 “this” 中的 “is”

（1）分组（()）

示例：(abc){2} 匹配 "abcabc"（而非 "abccc"）

示例：a|b 匹配 "a" 或 "b"；(abc)|(def) 匹配 "abc" 或 "def"

正则中部分字符有特殊含义（如 ./*/(），若需匹配其本身，需加 \ 转义。

错误类型 原因 解决方案

转义字符错误 未使用原始字符串，导致 \d 被解析为普通字符 所有正则规则都用原始字符串（r"表达式"）
贪婪匹配导致结果不符合预期 未加 ? 实现非贪婪匹配 在限定符后加 ?（如 *?/+?/{n,m}?）
分组提取失败 分组括号位置错误 检查 () 的范围，提取时用 group(n) 对应分组序号
中文匹配失败 未使用中文字符集 [\u4e00-\u9fa5] 匹配中文用 [\u4e00-\u9fa5]，避免用 .（. 不匹配中文）

import re

def validate_phone(phone):
    """验证手机号"""
    pattern = r"^1[3-9]\d{9}$"
    return bool(re.match(pattern, phone))

def validate_email(email):
    """验证邮箱"""
    pattern = r"^\w+@[a-zA-Z0-9]+\.[a-zA-Z]{2,}$"
    return bool(re.match(pattern, email))

def validate_pwd(pwd):
    """验证密码：8-16位，含字母+数字"""
    pattern = r"^(?=.*[a-zA-Z])(?=.*\d)[a-zA-Z\d]{8,16}$"
    return bool(re.match(pattern, pwd))

总结

核心要点

  1. 语法基础：掌握原子（\d/\w/[]）、限定符（*/+/{n,m}）、边界匹配（^/$）、分组（()）；
  2. 核心方法：
    • 开头匹配：match()；全文找第一个：search()；找所有：findall()；
    • 替换：sub()；分割：split()；复用规则：compile()；
  3. 实战技巧：
    • 用原始字符串 r"" 避免转义错误；
    • 贪婪/非贪婪匹配：*?/+? 实现最小匹配；
    • 标志位：re.I 忽略大小写，re.S 让 . 匹配换行；
  4. 避坑原则：
    • 复杂正则分步写（先测试原子，再加限定符，最后加分组）；
    • 优先复用成熟模板（如手机号、邮箱），避免重复造轮子；
    • 匹配中文用 [\u4e00-\u9fa5]，而非 .。

学习建议

  1. 先掌握核心语法和常用方法，再学习零宽断言等进阶内容；
  2. 用在线工具调试正则（如 Regex101），可视化匹配过程；
  3. 实际开发中，简单字符串操作（如替换固定字符）优先用字符串内置方法（str.replace()/str.split()），复杂场景再用正则。
